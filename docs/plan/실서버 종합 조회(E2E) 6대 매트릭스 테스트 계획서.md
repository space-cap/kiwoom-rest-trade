# 🛡️ 실서버 종합 조회(E2E) 6대 매트릭스 테스트 계획서

키움 REST API 전체 생성 코드의 안정성을 보증하기 위해, 국내/해외/계좌 영역과 단건/배열 데이터 유형을 대표하는 **6대 핵심 API를 선별하여 종합적인 실서버 연동 검증(E2E)**을 수행하기 위한 계획서입니다.

---

## 📊 E2E 6대 검증 매트릭스 구조

| 카테고리 | 단건 데이터 조회 (Single) | 배열/중첩 데이터 조회 (Array / Nested) |
|:---:|:---|:---|
| **국내주식** | **`ka10001`** (주식기본정보) <br> 👉 삼성전자 현재가 조회 (완료) | **`ka10003`** (주식일봉차트조회) <br> 👉 삼성전자 일봉 10일치 조회 (중첩 배열) |
| **미국주식** | **`oa10001`** (해외주식기본정보) <br> 👉 애플(AAPL) 현재가 조회 | **`oa10003`** (해외주식기간등락) <br> 👉 애플 차트 목록 조회 (중첩 배열) |
| **계좌/잔고** | **`kt00001`** (예수금현황요청) <br> 👉 대표 계좌 예수금 단건 조회 | **`kt00018`** (잔고요청) <br> 👉 보유 잔고 주식 목록 조회 (배열) |

---

## 🛠️ 세부 검증 시나리오 설계

1. **국내주식 일봉 차트 조회 (`ka10003`)**:
   * 삼성전자(005930)의 최근 일봉 데이터를 조회하여 Pydantic 중첩 모델(`List[Ka10003ResponseData]`) 역직렬화 및 한글 데이터 파싱을 검증합니다.

2. **미국주식 현재가 조회 (`oa10001`)**:
   * 애플(AAPL)의 현재가와 종목 정보를 조회하여 미국주식 게이트웨이(해외 도메인) 통신 및 해외 Rate Limit 딜레이 작동을 확인합니다.

3. **미국주식 차트 조회 (`oa10003`)**:
   * 애플(AAPL)의 기간등락 데이터를 조회하여 해외주식의 중첩 배열 모델이 오류 없이 파싱되는지 검증합니다.

4. **계좌 예수금 조회 (`kt00001`)**:
   * 계좌번호를 기입하지 않고 호출하여 로그인 세션 정보와 매핑된 사용자의 대표 계좌 예수금 총액 정보를 확인합니다.

5. **계좌 보유 잔고 조회 (`kt00018`)**:
   * 현재 계좌에 보유 중인 주식 종목들의 평단가, 수익률, 수량 등 배열 형태의 잔고 데이터가 에러 없이 조회되는지 확인합니다.
